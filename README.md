# 📘 Content Validator — Инструкция по эксплуатации

## 1. Назначение

**Content Validator** — утилита для технических специалистов, работающих с контентом XPLang: корреляциями, нормализациями, правилами обработки событий.

Функционал включает:

- преобразование отчётов **JSON → CSV**;
- обработку одиночных файлов и каталогов;
- автоматический запуск `evt-xp-linter` по всем уровням severity;
- генерацию JSON отчётов;
- массовую конвертацию JSON → CSV;
- визуальный прогресс выполнения и логирование.

---

## 2. Предварительные требования

### 2.1. Дистрибутив XPLang SDK

Необходимо скачать пакет, содержащий:

xp-sdk/
cli/
evt-xp-linter.exe


*(Ссылка для скачивания добавляется отдельно.)*

### 2.2. Комплект утилиты

Каталог должен содержать:

Content_Validator.exe
ContentValidator.ico
Reports/
Analyzed/


### 2.3. Права доступа

- Запуск исполняемых файлов.
- Чтение/запись в рабочие каталоги.

---

## 3. Запуск

Запуск выполняется запуском файла:

Content_Validator.exe


При первом старте автоматически создаются рабочие каталоги:

- `Reports`  
- `Analyzed`

---

## 4. Режимы работы

### 4.1 Один JSON файл

Назначение — конвертация одного JSON-отчёта в CSV.

Шаги:

1. Указать путь к JSON-файлу  
2. Указать путь для CSV  
3. Нажать **Запустить обработку**  

---

### 4.2 Каталог JSON файлов

Назначение — пакетная конвертация множества JSON-файлов.

Шаги:

1. Указать директорию JSON  
2. Указать директорию CSV  
3. Запустить обработку  

Каждый JSON будет преобразован в CSV.

---

### 4.3 Проверка контента

Основной режим. Предназначен для полной автоматизации анализа контента XPLang.

Функционал:

- запуск `evt-xp-linter` по severity:
  - `critical`
  - `error`
  - `warning`
  - `suggestion`
- генерация отдельных JSON отчётов;
- конвертация JSON → CSV;
- логирование stdout/stderr;
- отображение прогресса обработки.

Шаги:

1. Указать путь к SDK (`xp-sdk`)  
2. Указать директорию контента (`knowledgebase_YYYY_MM_DD`)  
3. Указать каталог JSON отчётов (по умолчанию `Reports`)  
4. Указать каталог CSV (по умолчанию `Analyzed`)  
5. Запустить  

Выходные файлы:

Reports/critical_YYYY_MM_DD.json
Reports/error_YYYY_MM_DD.json
Reports/warning_YYYY_MM_DD.json
Reports/suggestion_YYYY_MM_DD.json
Analyzed/*.csv


Дата извлекается автоматически из имени каталога контента.

---

## 5. Особенности

### 5.1. Многопоточность

GUI остаётся отзывчивым — вычисления выполняются в рабочих потоках.

### 5.2. Прогрессбар

Отображает процесс выполнения:

- 4 шага линтера  
- 1 шаг конвертации JSON → CSV  

### 5.3. Логирование

Показывает:

- запущенные команды  
- stdout / stderr линтера  
- предупреждения  
- информацию о сформированных CSV  

### 5.4. Автоматическое создание каталогов

При старте создаются:

Reports/
Analyzed/


### 5.5. Поддержка EXE

Утилита корректно работает как в режиме `.py`, так и в собранном `.exe`.

---

## 6. Диагностика типичных ошибок

### ❗ evt-xp-linter.exe не найден  
Неверно указан путь к SDK.

### ❗ JSON отчёт пустой  
В выбранном severity нет объектов.  
Проверить taxonomy и содержимое контента.

### ❗ CSV не созданы  
JSON отчёты пустые и автоматически удаляются.

---

## 7. Итог

**Content Validator** — инструмент автоматизации анализа, валидации и преобразования контента XPLang.  
Предназначен для инженеров SOC, разработчиков SIEM-контента и специалистов по XPLang.

Утилита обеспечивает:

- автоматическую проверку контента;
- генерацию отчётов JSON;
- массовую конвертацию JSON → CSV;
- удобный GUI с прогрессом и логированием;
- автономную работу в формате единого исполняемого файла.
